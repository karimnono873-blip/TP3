1. Contexte Théorique et Objectifs du TP

L'objectif principal de ce troisième Travail Pratique est de mettre en œuvre les concepts fondamentaux de la Programmation Orientée Objet (POO) à travers un langage fortement typé tel que Java. Le cas d'étude choisi est le jeu classique du "Saute-Mouton", un problème algorithmique qui nécessite une gestion rigoureuse de l'état d'un système (ici, modélisé par un tableau unidimensionnel) et l'application de contraintes de déplacement strictes.

1.1. Rappel sur la gestion de la mémoire (Tableaux en Java)

Contrairement au C++ où la taille d'un tableau statique doit être connue à la compilation, Java permet une allocation dynamique à l'exécution via le mot-clé new. Dans notre contexte, la taille du plateau de jeu est déterminée par l'utilisateur au moment de l'initialisation. La déclaration char[] Plateau; crée uniquement une référence. L'instanciation réelle s'effectue avec Plateau = new char[2 * nbPions + 1];, allouant un espace contigu en mémoire dans le "Heap" (tas). Cette flexibilité est cruciale pour respecter le cahier des charges du TP.

1.2. Formalisation des Règles du Jeu

Le système est défini par un automate à états finis. Les règles de transition (les "coups") sont contraintes :

  • Asymétrie directionnelle : Les pions Noirs ('N') ne progressent que vers l'index supérieur (droite), tandis que les pions Rouges ('R') ne progressent que vers l'index inférieur (gauche).
  • Déplacement simple : Un pion peut avancer d'une case (Δ = ±1) si et seulement si l'index cible correspond à l'espace vide.
  • Saut par-dessus un obstacle : Un pion peut effectuer un bond (Δ = ±2) si la case cible est vide ET que la case intermédiaire (Δ = ±1) contient un pion de la faction opposée.
  • Condition d'arrêt : Le jeu se termine soit par un succès (la configuration initiale est parfaitement inversée), soit par un blocage matériel (aucune transition valide n'est possible pour l'ensemble des pions restants).

2. Implémentation : Code Source Java

Le code suivant représente la solution monolithique répondant strictement à l'arbre algorithmique fourni dans l'énoncé du TP. Il intègre les sous-programmes obligatoires : initialisation, affichage, exécution d'un coup, et vérification de la fin de jeu. 

import java.util.Scanner;

/**
 * Classe principale résolvant le problème du Saute-Mouton.
 * Auteur : Dahane Ahmed Lamine (M1 AII)
 */
public class SauteMouton {
    
    // Déclaration des variables globales selon les exigences de M. Menani
    static final int NbMAX = 10; // Nombre maximum de pions par couleur
    static char[] Plateau;         // Le plateau modélisant le système
    static int nbPions;            // Nombre de pions par couleur choisi par l'utilisateur
    static int indVide;            // Pointeur constant vers l'index de la case vide

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        boolean rejouer = true;

        // Boucle principale du programme (Permet de relancer une partie)
        while (rejouer) {
            initialiserPlateau(scanner);
            boolean fini = false;

            // Boucle d'exécution d'une partie (Automate)
            while (!fini) {
                afficherPlateau();
                jouerCoup(scanner);
                fini = finDeJeu();
            }

            System.out.print("\nVoulez-vous refaire une partie ? (O/N) : ");
            char rep = scanner.next().toUpperCase().charAt(0);
            if (rep != 'O') rejouer = false;
        }
        scanner.close();
        System.out.println("Fin du programme.");
    }

    /**
     * Module 1 : Initialisation dynamique du système
     */
    static void initialiserPlateau(Scanner scanner) {
        do {
            System.out.print("Entrez le nombre de pions (max " + NbMAX + ") : ");
            nbPions = scanner.nextInt();
        } while (nbPions <= 0 || nbPions > NbMAX);

        // Allocation dynamique (N pions noirs + 1 espace + N pions rouges)
        Plateau = new char[2 * nbPions + 1];
        
        // Remplissage de la moitié gauche avec 'N'
        for (int i = 0; i < nbPions; i++) Plateau[i] = 'N';
        
        // Positionnement de la case vide au centre
        indVide = nbPions;
        Plateau[indVide] = ' ';
        
        // Remplissage de la moitié droite avec 'R'
        for (int i = nbPions + 1; i < 2 * nbPions + 1; i++) Plateau[i] = 'R';
    }

    /**
     * Module 2 : Affichage de l'état actuel de la mémoire (Rendu visuel)
     */
    static void afficherPlateau() {
        System.out.println("\nPlateau actuel :");
        for (int i = 0; i < Plateau.length; i++) {
            System.out.print("[" + Plateau[i] + "]");
        }
        System.out.println();
        // Affichage des index pour guider l'utilisateur
        for (int i = 0; i < Plateau.length; i++) {
            System.out.print(" " + i + " ");
        }
        System.out.println();
    }

    /**
     * Module 3 : Logique de transition d'état et vérification des contraintes
     */
    static void jouerCoup(Scanner scanner) {
        System.out.print("Index du pion à déplacer : ");
        int pos = scanner.nextInt();

        // Sécurité 1 : Out of Bounds Exception Prevention
        if (pos < 0 || pos >= Plateau.length || pos == indVide) {
            System.out.println("Erreur : Sélection invalide.");
            return;
        }

        char pion = Plateau[pos];

        // Arbre de décision selon l'algorithme du PDF
        if (pion == 'R') {
            if (pos - 1 == indVide) {
                // Cas 1 : Déplacement simple à gauche
                echanger(pos, indVide);
            } else if (pos - 2 == indVide && Plateau[pos - 1] == 'N') {
                // Cas 2 : Saut par-dessus un Noir vers la gauche
                echanger(pos, indVide);
            } else {
                System.out.println("Règle violée : Coup interdit pour le Rouge.");
            }
        } else if (pion == 'N') {
            if (pos + 1 == indVide) {
                // Cas 1 : Déplacement simple à droite
                echanger(pos, indVide);
            } else if (pos + 2 == indVide && Plateau[pos + 1] == 'R') {
                // Cas 2 : Saut par-dessus un Rouge vers la droite
                echanger(pos, indVide);
            } else {
                System.out.println("Règle violée : Coup interdit pour le Noir.");
            }
        }
    }

    /**
     * Sous-module utilitaire : Swap en mémoire
     */
    static void echanger(int p1, int p2) {
        char temp = Plateau[p1];
        Plateau[p1] = Plateau[p2];
        Plateau[p2] = temp;
        // Mise à jour critique du pointeur de la case vide
        indVide = p1; 
    }

    /**
     * Module 4 : Analyse de l'état final (Victoire ou Deadlock)
     */
    static boolean finDeJeu() {
        // Scan de Victoire : Les Rouges ont-ils envahi la partie gauche ?
        boolean gagne = true;
        for (int i = 0; i < nbPions; i++) {
            if (Plateau[i] != 'R') gagne = false;
        }
        if (gagne) {
            System.out.println("*** SUCCÈS : Configuration finale atteinte ! ***");
            return true;
        }

        // Scan de Blocage (Deadlock Detection Algorithm)
        boolean bloque = true;
        for (int i = 0; i < Plateau.length; i++) {
            if (i == indVide) continue; // Ignorer l'espace vide
            
            // Existe-t-il au moins UN pion rouge pouvant bouger ?
            if (Plateau[i] == 'R' && (i - 1 == indVide || (i - 2 == indVide && Plateau[i - 1] == 'N'))) {
                bloque = false; break;
            }
            // Existe-t-il au moins UN pion noir pouvant bouger ?
            if (Plateau[i] == 'N' && (i + 1 == indVide || (i + 2 == indVide && Plateau[i + 1] == 'R'))) {
                bloque = false; break;
            }
        }

        if (bloque) {
            System.out.println("!!! ECHEC : Situation de blocage détectée. !!!");
            return true;
        }
        
        // Le jeu continue
        return false;
    }
}

2.1. Explications et Analyse Algorithmique

Le code ci-dessus, bien que procédural dans sa syntaxe interne, pose les bases de la création d'une classe d'objet `SauteMouton`. Chaque méthode (`initialiserPlateau`, `jouerCoup`, etc.) représente un comportement encapsulé du système.

Analyse de la complexité : La méthode la plus coûteuse en temps d'exécution est `finDeJeu()`. Lors de la détection de blocage, elle doit parcourir le tableau de taille `T = 2N + 1`. La complexité temporelle est donc de l'ordre de O(N), où N est le nombre de pions par équipe. L'échange de pions dans `echanger()` se fait en temps constant O(1) grâce à la variable temporaire. La complexité spatiale est également de O(N), ce qui garantit une exécution extrêmement rapide même pour des valeurs de N très élevées (bien que le TP limite N à 10 pour des raisons de jouabilité humaine).
